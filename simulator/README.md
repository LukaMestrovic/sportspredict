# Bundled Simulator

This directory is the numerical simulator used as optional context by the
pricing LLM. It is deliberately isolated from `bot/`: the live bot stays
standard-library plus `requests`, while the simulator runs in a child Python
process with its own dependency list.

Everything required to operate the simulator at runtime is tracked here:

- `src/sportspredict/` contains the baseline generative match engine.
- `src/sphybrid/` contains learned-rate, event-timing, player-allocation,
  training, backtest, validation, and report code.
- `config/` contains the model and market contract configuration.
- `data/` contains the fitted model, compact lookup tables, Elo snapshot, and
  frozen validation evidence used in audits. The validation artifact includes
  family-level rolling-origin and WC2026 Brier comparisons against always-50%
  and prior exact-contract empirical-rate baselines, with match-count warnings.

The maintained training pipeline is also tracked here. The key commands are:

```bash
PYTHONPATH=simulator/src SPORTSPREDICT_ROOT=simulator \
  python -m sphybrid.cli train

PYTHONPATH=simulator/src SPORTSPREDICT_ROOT=simulator \
  python -m sphybrid.cli ingest --limit 10

PYTHONPATH=simulator/src SPORTSPREDICT_ROOT=simulator \
  python -m sphybrid.cli backtest --in-sample --n-sims 500
```

The default train command reads `data/processed/history_stat_table.parquet` and
refreshes the production artifacts `rate_model.joblib`, `rate_model.json`, and
`team_ratings.json`. Player-share fitting reads
`data/processed/player_stat_table.parquet` and writes the compact
`player_shares.json` runtime artifact.

What is intentionally excluded: exploratory notebooks, raw API-Football caches,
historical odds archives, generated review CSVs, rendered PDF reports, virtual
environments, bytecode caches, and standalone competition clients. The checked-in
processed tables are compact inputs for reproducing the fitted artifacts; raw
fetch caches can be regenerated by the ingestion commands when credentials are
available.
